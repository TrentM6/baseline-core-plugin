# Ideation Methods

> Structured techniques for generating, evaluating, and converging on design ideas.

**Part of:** [UX Design](../ux-design-skill.md)

---

## Design Studio Method

### What It Is

A structured group exercise where team members individually sketch solutions, then present and critique them as a group. Combines individual creative thinking with collaborative refinement.

### Process

**1. Brief (10 min):**
Present the design challenge clearly:
- The user problem to solve
- Key constraints (technical, business, user)
- What a good solution looks like (success criteria)

**2. Individual sketching — Round 1 (10 min):**
- Everyone sketches 3-4 concepts independently
- Sketches are rough — ideas, not designs
- No talking during sketching

**3. Present and critique (3 min per person):**
- Each person presents their sketches
- Group gives structured feedback (plus/delta)
- Note which ideas or elements resonate across sketches

**4. Individual sketching — Round 2 (10 min):**
- Everyone sketches again, incorporating ideas from Round 1
- Can evolve their own concept or adopt elements from others
- Focus on the most promising direction

**5. Share and converge (15 min):**
- Present Round 2 sketches
- Dot vote on strongest concepts or elements
- Identify the design direction to take forward

### When to Use

- Multiple valid approaches to a design problem
- Need team alignment on a direction
- Cross-functional input improves the solution (engineering, product, design together)
- Kick-starting a new feature or redesign

## Crazy 8s

### What It Is

Rapid ideation: sketch 8 ideas in 8 minutes (1 minute per idea). Forces speed and volume over perfection.

### Process

1. Fold a sheet of paper into 8 panels
2. Set a timer for 8 minutes
3. Sketch one idea per panel, one minute each
4. When time is up, stop (even if not all panels are filled)
5. Review: circle the strongest 1-2 ideas
6. Share with the group

### Rules

- Speed over quality — ugly sketches are fine
- Each panel should be a distinct idea (not variations of the same thing)
- No erasing — if an idea isn't working, move to the next panel
- It's okay if later panels are weaker — the first few are often the obvious ideas, and the last few push into creative territory

### When to Use

- Early ideation when you need volume
- Warm-up exercise before a design studio
- Breaking out of a rut when everyone is fixated on one approach
- Any time you need to generate many ideas fast

## "How Might We" Framing

### What It Is

A reframing technique that turns problems, insights, and observations into opportunity questions. Opens up solution space without constraining to a specific approach.

### Format

```
How might we [desired outcome] for [user] when [context/constraint]?
```

### Generating HMWs

**From a user problem:**
- Problem: "Users can't find the search feature"
- HMW: "How might we make search discoverable without cluttering the interface?"

**From a research insight:**
- Insight: "Users create spreadsheets to track things our product already tracks"
- HMW: "How might we surface tracking features so users don't need workarounds?"

**From a constraint:**
- Constraint: "We can't add more navigation items"
- HMW: "How might we help users access more features within the existing navigation?"

### Quality Check

**Too broad:** "How might we make the product better?" → Can't design for this
**Too narrow:** "How might we add a search bar to the header?" → This is already a solution
**Just right:** "How might we help users find features quickly?" → Opens up multiple design approaches

### When to Use

- After research synthesis, before ideation
- When the team is stuck on a problem framing
- To turn constraints into creative opportunities
- As voting/prioritization input: which HMW should we solve first?

## SCAMPER Method

### What It Is

A structured creativity technique that applies seven prompts to an existing product or feature to generate new ideas.

### The Seven Prompts

| Prompt | Question | Example |
|--------|----------|---------|
| **Substitute** | What can we replace? | Replace manual data entry with auto-detection |
| **Combine** | What can we merge? | Combine search and filter into one unified interface |
| **Adapt** | What can we borrow from elsewhere? | Adapt the swipe gesture from mobile to desktop |
| **Modify** | What can we change in scale, shape, or form? | Make the dashboard customizable instead of fixed |
| **Put to other use** | What else could this be used for? | Use the activity feed for onboarding education |
| **Eliminate** | What can we remove? | Remove the confirmation step in low-risk actions |
| **Reverse** | What if we did the opposite? | Instead of users finding content, content finds users |

### Process

1. Pick a feature or experience to improve
2. Walk through each SCAMPER prompt (5 min per prompt)
3. Write down every idea, even bad ones
4. Review all ideas, pick the most promising
5. Develop top ideas into concepts

### When to Use

- Improving an existing feature (not creating from scratch)
- Generating incremental innovation ideas
- Breaking habitual thinking about a familiar product area
- Quick ideation session when you need structured creativity

## Brainwriting

### What It Is

Silent, parallel idea generation. Everyone writes ideas simultaneously and builds on others' ideas. Avoids the groupthink and anchoring bias of traditional brainstorming.

### Process (6-3-5 Method)

1. **6 people** each write **3 ideas** in **5 minutes**
2. Pass your sheet to the person next to you
3. Read their ideas, then write 3 more ideas inspired by (or building on) theirs
4. Repeat until everyone has contributed to every sheet
5. Review all ideas as a group

### Rules

- No talking during writing rounds
- Build on others' ideas, don't just add unrelated ones
- Write clearly enough for others to read
- Every idea is valid — no filtering during generation

### Why It's Better Than Brainstorming

- **No anchoring:** First idea doesn't set the frame for everyone
- **Equal participation:** Quiet people contribute as much as loud people
- **More ideas:** Parallel generation produces more output in less time
- **Building:** Ideas evolve as they pass through different perspectives

### When to Use

- Groups where some members are quiet or junior
- Topics where the first idea tends to dominate discussion
- When you need sheer volume of ideas
- Remote-friendly (everyone writes in a shared doc simultaneously)

## Concept Sketching

### What It Is

Communicating design ideas visually at a level that's intentionally rough. Not about drawing skill — about communicating the idea clearly.

### What to Include

| Element | Purpose | Example |
|---------|---------|---------|
| Screen layout | Shows structure and hierarchy | Boxes for content areas, rectangles for buttons |
| Key UI elements | Shows what's on screen | Search bar, navigation, cards, forms |
| Annotations | Explains behavior | "Tapping here opens a modal" |
| Flow arrows | Shows sequence | "Screen A → Screen B when user clicks Submit" |
| User/actor | Shows who's involved | Stick figure with label |

### What NOT to Include

- Pixel-perfect details (save for later)
- Color (unless it's communicating meaning)
- Real content (placeholder text is fine)
- Visual polish of any kind

### Tips

- Use thick markers — prevents over-detailing
- Time-box: 2-5 minutes per sketch
- Draw multiple screens to show flow, not just a single screen
- Add annotations — your sketch should be understandable without you explaining it
- Take photos immediately — sketches on whiteboards get erased

## Idea Evaluation and Convergence

### Dot Voting

**Process:**
1. Display all ideas visually (wall, board, shared doc)
2. Each person gets 3-5 dots (stickers or digital votes)
3. Vote silently — place dots on your preferred ideas
4. Count votes to identify the most popular ideas

**Rules:**
- Can place multiple dots on one idea (if you feel strongly)
- Vote first, discuss after (prevents influence)
- Dot voting is input, not a final decision — discuss the top-voted ideas before committing

### Feasibility/Impact Scoring

Plot ideas on a 2x2 matrix:

```
High Impact
      |
  PRIORITIZE          DO FIRST
  (High Impact,       (High Impact,
   Low Feasibility)    High Feasibility)
      |
  SKIP                QUICK WINS
  (Low Impact,        (Low Impact,
   Low Feasibility)    High Feasibility)
      |
Low Impact
      |
Low Feasibility —————— High Feasibility
```

### Concept Testing Criteria

Before committing to a concept, evaluate against:

| Criteria | Question |
|----------|----------|
| **Desirability** | Do users want this? Is there evidence of the need? |
| **Feasibility** | Can we build this? Do we have the technology and skills? |
| **Viability** | Does this make business sense? Does it align with strategy? |
| **Usability** | Can users figure out how to use this? |
| **Differentiation** | Does this set us apart from alternatives? |

## When to Diverge vs. Converge

### The Double Diamond in Practice

```
    Discover          Define          Develop          Deliver
   (Diverge)        (Converge)      (Diverge)        (Converge)

   Research          Insights        Ideation          Final
   Explore           Problem         Concepts          Solution
   Options           Statement       Prototypes        Ship
```

### Diverge Phase Rules

- Generate quantity — more ideas, more options
- Defer judgment — no "yes but" during ideation
- Build on others' ideas — "yes and"
- Seek wild ideas — they often contain seeds of great solutions
- Stay visual — sketch, don't just talk

### Converge Phase Rules

- Apply evaluation criteria (not just gut feel)
- Make choices and commit — convergence requires killing ideas
- Be explicit about why you're choosing (document the rationale)
- Test the chosen direction before investing heavily

### Common Mistakes

**Converging too early:**
- Symptom: Team latches onto the first decent idea
- Fix: Require minimum idea count before evaluating (e.g., "We need 10 concepts before we vote")

**Never converging:**
- Symptom: Endless exploration, no decisions, no progress
- Fix: Set a deadline for convergence. "By Friday, we'll have one direction."

**Skipping divergence:**
- Symptom: Jumping straight to "the solution" without exploring alternatives
- Fix: Even if you think you know the answer, spend 30 minutes exploring alternatives. You'll either confirm your instinct or find something better.

# Visual Communication

Visual communication is the discipline of making complex ideas visible and persuasive. It spans presentations, diagrams, data visualization, and product narratives — turning strategy, data, and decisions into visual stories that drive alignment and action.

## Core Principles

1. **One idea per slide/diagram.** If you need two ideas, make two visuals.
2. **Narrative over data dump.** Structure a story. Don't just present information.
3. **Visual hierarchy drives attention.** Size, color, position tell the eye where to go.
4. **Less is more.** Remove everything that doesn't serve the message.
5. **Design for the room.** Presentation context (live, async, print) changes everything.
6. **Data tells, story sells.** Numbers need narrative context to drive decisions.

## Workflow

### 1. Clarify Before Starting
- What's the purpose (inform, persuade, align, decide)?
- Who is the audience?
- What's the context (live, async, print)?
- What's the key message?
- What data/content exists?

### 2. Choose the Right Format

| Need | Format |
|------|--------|
| Persuade/align stakeholders | Narrative presentation |
| Explain a process | Flowchart or diagram |
| Show data patterns | Data visualization |
| Map a system | Architecture or systems diagram |
| Support a decision | Decision matrix or comparison |

### 3. Design Process

**Structure:** Define the narrative arc > outline key points > choose visual format > sketch layouts.

**Create:** Build slides/diagrams > apply visual hierarchy > add data visualizations > review for clarity.

**Refine:** Simplify > check one-idea-per-slide > test with audience > iterate.

## Quality Checks

- [ ] Clear narrative thread?
- [ ] One idea per visual?
- [ ] Visual hierarchy guides the eye?
- [ ] Consistent with visual identity?
- [ ] Works in the delivery context?
- [ ] Data accurately represented?
- [ ] Audience-appropriate complexity?

## Anti-Patterns

| Anti-Pattern | Instead |
|--------------|---------|
| Slide decks as documents | Slides for presenting, docs for reading |
| Data dump without narrative | Structure a story around the data |
| Inconsistent styling | Follow visual identity standards |
| Too much text per slide | One idea, minimal text, strong visual |
| Decorative graphics | Every visual element should communicate |
